# 장기 규칙 명세 초안

작성일: 2026-04-21

## 목적

- 게임 엔진 구현 전에 규칙 해석 차이를 줄인다.
- 서버/클라이언트/UI 테스트가 동일한 규칙 문서를 기준으로 동작하게 한다.

## 우선 결정할 항목

- 한/초 진영 표기 기준 확정
- 선공/후공 기본값 확정
- 빅장, 장군 반복, 무승부 처리 정책 확정
- 시간 제한 및 시간패 규칙 포함 시점 확정

## 보드와 좌표

- 보드 크기: 9 x 10
- 좌표 체계: 좌상단 `(0, 0)` 기준 사용 여부 확정
- 궁성 좌표를 명시적으로 문서화

## 기물 정의

- 장
- 사
- 상
- 마
- 차
- 포
- 졸

각 기물별로 아래 항목을 확정한다.

- 시작 위치
- 이동 가능 경로
- 경로 차단 조건
- 상대 기물 포획 조건
- 궁성 내부 예외 규칙

## 턴과 수 처리

- 한 턴에 한 번의 합법 수만 허용
- 현재 플레이어는 자가 장군 상태를 만드는 수를 둘 수 없음
- 서버가 최종 수 검증 권한을 가짐

## 승패 조건

- 외통 판정 조건
- 기권 처리
- 시간패 처리 여부
- 무승부/재대국 처리 여부

## 테스트 케이스 기준

- 기물별 합법 이동 케이스
- 기물별 불법 이동 케이스
- 체크 유발/해제 케이스
- 자가 장군 금지 케이스
- 승패 판정 케이스

## 구현 연결 메모

- `packages/shared-types`에서 좌표, 기물, 게임 상태 타입 관리
- `packages/game-engine`에서 순수 함수 기반 규칙 검증 구현
- 테스트 우선으로 규칙을 잠근 뒤 UI에 연결
