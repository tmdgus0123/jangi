# 장기 규칙 정리 보고서 (엔진 디버깅용)

작성일: 2026-04-21
목적: `장`이 기물을 포획하지 못하는 문제를 재발 없이 해결하기 위한 규칙 기준을 명확히 한다.

## 1. 기준 좌표와 보드

- 보드 크기: 9 x 10
- 좌표: 좌상단을 `(0, 0)`, 우하단을 `(8, 9)`로 사용
- 궁성(3 x 3)
  - 한 궁(상단): `x in [3..5]`, `y in [0..2]`
  - 초 궁(하단): `x in [3..5]`, `y in [7..9]`
- 궁성 대각선 연결(실제 선이 있는 경로만 허용)
  - 상단 궁: `(3,0) <-> (4,1) <-> (5,2)`, `(5,0) <-> (4,1) <-> (3,2)`
  - 하단 궁: `(3,7) <-> (4,8) <-> (5,9)`, `(5,7) <-> (4,8) <-> (3,9)`

## 2. 기물 공통 규칙

- 자기 말이 있는 칸으로는 이동/포획 불가
- 상대 말이 있는 칸은 해당 기물 이동 규칙이 허용하면 포획 가능
- 수를 둔 뒤 자기 `장`이 장군 상태가 되면 그 수는 불법(자가 장군 금지)

## 3. `장`(왕) 규칙 (핵심)

- 이동 범위: 궁성 내부에서 1칸
- 가능한 방향
  - 상하좌우 1칸
  - 궁성 대각선이 있는 칸들 사이에서만 대각 1칸
- 불가능한 이동
  - 궁성 바깥으로 이동
  - 궁성에 선이 없는 대각선 이동
- 포획 규칙
  - 위 이동 규칙으로 도달 가능한 칸에 상대 기물이 있으면 포획 가능
  - 즉, `장`은 "이동 가능한 칸"이면 일반 기물처럼 포획할 수 있어야 함

## 4. 장군/외통 관련 최소 규칙

- 장군(Check): 현재 턴이 아닌 쪽의 `장`이 상대의 다음 합법 공격 범위에 포함된 상태
- 합법 수 필터: 후보 수를 적용했을 때 내 `장`이 장군이면 제거
- 외통(Checkmate): 장군 상태 + 합법 수가 0개

## 5. `장` 포획 불가 버그가 생기는 대표 원인

- 궁성 대각선을 "모든 대각 1칸"으로 처리하거나 반대로 "대각을 전부 금지"한 경우
- 이동 생성 단계에서는 포획을 만들었지만, 합법 수 필터에서 잘못 제거한 경우
- 수 적용 후 "장끼리 마주봄" 또는 장군 판정 로직이 과도하게 적용되어 정상 포획 수까지 제거한 경우
- 좌표/진영 기준 혼동으로 궁성 범위를 반대로 계산한 경우

## 6. 엔진 구현 체크리스트

- `장` 이동 생성
  - 궁성 내부 확인
  - 대각은 궁성 선 연결 테이블로 허용 여부 확인
  - 목적지에 아군 말이 있으면 제외
  - 목적지에 적군 말이 있으면 포획 수 생성
- 합법 수 필터
  - 수를 적용한 보드에서 내 `장` 위치를 다시 찾는다
  - 내 `장`이 공격받으면 해당 수 제거
- 판정 순서
  - 이동 생성 -> 후보 수 적용 -> 자가 장군 검사 -> 합법 수 확정

## 7. 권장 회귀 테스트 (필수)

- `장`이 상하좌우 포획 가능한 케이스 4개
- `장`이 궁성 대각선 위에서만 포획 가능한 케이스
- 궁성 대각선이 없는 칸으로의 대각 이동/포획이 거부되는 케이스
- 포획 직후 자가 장군이 되면 해당 수가 불법 처리되는 케이스
- 포획 직후에도 자가 장군이 아니면 합법 처리되는 케이스

## 8. 결론

`장`이 기물을 못 먹는 문제는 대부분 "이동 생성 규칙" 자체보다는,
"궁성 대각선 허용 조건" 또는 "자가 장군 필터" 단계에서 발생한다.

따라서 수정 우선순위는 아래 순서를 권장한다.

1. 궁성 대각선 연결을 좌표 테이블로 고정
2. `장` 포획 수 생성 여부를 단위 테스트로 고정
3. 합법 수 필터(자가 장군 검사) 전/후 결과를 테스트에서 분리 검증
